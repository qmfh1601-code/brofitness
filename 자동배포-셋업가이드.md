# 브로 저널 — 예약발행 + 자동배포 셋업 가이드

> 목표: **칼럼을 미리 써두고 날짜만 정해두면, 그날 자동으로 라이브에 올라간다.**
> 맥북이 꺼져 있어도 GitHub 클라우드가 매일 알아서 발행한다.

이미 코드/설정은 다 준비됨. 아래 **최초 1회 연결**만 끝내면 완성입니다.
(②③은 사용자 로그인이 필요해 직접 해야 하는 부분 — 차근차근 따라오시면 5~10분.)

---

## 한눈에 보는 구조

```
data/columns.json (글 + 발행날짜)   ←  여기만 수정하면 됨
        │
   git push  (배포.command 더블클릭)
        ▼
   GitHub 저장소  ──────────────┐
        │                      │ 매일 00:10 (자동, 맥 꺼져도 됨)
        │ push 때마다           │ GitHub Actions가 재빌드 신호
        ▼                      ▼
   Netlify 가 publish.py 실행 → "오늘 날짜가 된 글"만 공개 → https://brofitness.kr
```

---

## ① (완료) 코드·설정 준비됨
- `tools/publish.py` : 발행일이 **오늘(한국시간) 이하**인 글만 공개. 미래 날짜 글은 자동 대기.
- `netlify.toml` : Netlify가 배포 때마다 `publish.py`를 돌리도록 설정.
- `.github/workflows/daily-publish.yml` : 매일 00:10 KST 자동 발행.
- `배포.command` : 더블클릭 = 재생성 + GitHub 푸시.
- 사진: 글에 `image`를 안 쓰면 카테고리에 맞는 우리 사진 자동 배정 + 본문에도 한 장 자동 삽입.

## ② GitHub에 올리기 (최초 1회)

1. https://github.com 로그인 (구글 계정 qmfh1601 로 가입/로그인).
2. 오른쪽 위 **＋ → New repository**
   - Repository name: `brofitness`
   - **Private** 선택 (공개 안 함)
   - 나머지 체크박스 **다 끄고** (README 추가 X) → **Create repository**
3. 생성 후 나오는 화면에서 `…or push an existing repository` 쪽 주소를 복사.
   대개 `https://github.com/<내아이디>/brofitness.git`
4. 터미널(또는 저한테 말씀)에서 아래 실행:
   ```bash
   cd ~/bro-sis-web
   git branch -M main
   git remote add origin https://github.com/<내아이디>/brofitness.git
   git push -u origin main
   ```
   - 비밀번호를 물으면 GitHub 비밀번호가 아니라 **Personal Access Token**이 필요할 수 있어요.
     GitHub → Settings → Developer settings → Personal access tokens → **Generate new token (classic)**,
     `repo` 권한 체크 → 생성된 토큰을 비밀번호 자리에 붙여넣기.
   - (이 부분 막히면 저를 부르세요. 같이 처리합니다.)

## ③ Netlify를 GitHub에 연결 (수동 드래그 → 자동으로 전환)

> 도메인(brofitness.kr)·SSL은 **그대로 유지**됩니다. 배포 '소스'만 드래그에서 GitHub로 바꾸는 것.

1. https://app.netlify.com 로그인 → 기존 프로젝트 **brofitness.kr** 클릭.
2. **Site configuration → Build & deploy → Continuous deployment → Link repository**
   (또는 "Link site to Git") 클릭.
3. **GitHub** 선택 → 권한 승인 → 방금 만든 `brofitness` 저장소 선택.
4. 빌드 설정은 `netlify.toml`이 자동으로 채워줍니다. 확인만:
   - Branch: `main`
   - Build command: `python3 tools/publish.py`
   - Publish directory: `.`
   → **Deploy** / 저장.
5. 첫 자동 배포가 돌고 https://brofitness.kr 가 그대로 뜨면 성공.
   → 이제부터 **GitHub에 push할 때마다 자동 배포**. (수동 드래그 더 이상 안 해도 됨)

### 빌드 후크 만들기 (매일 자동발행용)
6. 같은 화면 **Build & deploy → Build hooks → Add build hook**
   - 이름: `daily` / Branch: `main` → 저장.
   - 생성된 URL(`https://api.netlify.com/build_hooks/xxxx`) **복사**.
7. GitHub 저장소로 가서 **Settings → Secrets and variables → Actions → New repository secret**
   - Name: `NETLIFY_BUILD_HOOK`
   - Secret: 방금 복사한 URL 붙여넣기 → 저장.

끝! 이제 매일 00:10(KST)에 GitHub가 Netlify를 깨워 그날 글을 공개합니다.

## 평소 사용법 (셋업 끝난 뒤)

### A. 새 칼럼 예약발행
1. `data/columns.json` 의 `posts` 맨 위에 새 글 블록 추가:
   ```json
   {
     "id": "summer-diet-2026",
     "cat": "다이어트",
     "title": "여름 전 4주, 현실적인 다이어트 플랜",
     "date": "2026.07.01",          // ← 이 날짜가 되면 자동 공개 (미래로 적어두면 예약)
     "excerpt": "한 줄 요약 (검색결과에 보임)",
     "body": [
       "첫 문단…", "둘째 문단…", "셋째 문단…", "넷째 문단…"
     ]
   }
   ```
   - `image` 는 **안 써도 됨** → 카테고리에 맞는 우리 사진 자동 배정 + 본문에도 자동 한 장.
   - 특정 사진을 쓰고 싶으면 `"image": "img/yongam-2.jpg"`.
   - 본문 원하는 위치에 사진: `body` 배열에 `{"img":"img/gym3.jpg","caption":"용암점 머신존"}` 끼우기.
2. **`배포.command` 더블클릭** → GitHub에 올라가고 Netlify 자동 배포.
   - 미래 날짜 글은 그날까지 안 보이다가, 발행일 새벽에 자동 공개.
   - 여러 편을 한꺼번에 써서 날짜만 다르게 박아두면 → 매일/매주 알아서 한 편씩 올라감.

### B. 지금 당장 올리고 싶을 때
`배포.command` 더블클릭 (날짜가 오늘 이하면 즉시 공개).

### C. 발행 예약 현황 확인
터미널에서 `python3 tools/publish.py` 실행 → 맨 아래 "⏳ 예약 대기" 목록에 미래 글이 표시됨.

## 자주 묻는 것
- **맥북을 꺼놔도 자동발행 되나요?** → 네. 매일 발행은 GitHub 클라우드에서 돌아갑니다.
- **수동 드래그 방식은 이제 안 쓰나요?** → 안 써도 됩니다(백업으로 남겨둠). `배포.command`가 대체.
- **발행 시각을 바꾸고 싶어요** → `.github/workflows/daily-publish.yml` 의 `cron: "10 15 * * *"` 줄 수정
  (UTC 기준. 한국시간 = UTC+9. 예: 매일 오전 7시 KST = `0 22 * * *`).
- **글을 내리고 싶어요** → 해당 글 블록에 `"draft": true` 추가 후 배포.
